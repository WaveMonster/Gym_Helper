\# 🏋️‍♂️ Gym\_Helper (核心力量训练追踪引擎)



\*\*Stop guessing your weights. Let the algorithm dictate your gains.\*\*

(别再瞎猜今天推多重了，让算法来决定你的训练容量。)



这是一个基于 Python 开发的轻量级命令行交互程序 (CLI)。它专为力量训练爱好者设计，通过内置的 \*\*三重递进模型 (Triple Progression Method)\*\* 状态机，严格管理并计算你每一次训练的重量、次数和组数。它可以有效防止盲目加重导致的受伤，或长期使用同一重量导致的平台期。



\## ✨ 核心特性 (Features)



\* \*\*🧠 智能状态机 (ABC 计划)：\*\* 自动根据你上一次的训练表现，决定今天是该加重量（状态 A）、加次数（状态 B）还是加组数（状态 C）。

\* \*\*⚙️ 动态步长适配：\*\* 完美适配健身房的不同器械。无论是 2.5kg 一档的杠铃，还是 5kg 一档的绳索下拉，都能精准推算下次目标。

\* \*\*🗄️ 本地数据安全：\*\* 拒绝云端隐私泄露。所有训练档案、容量数据均以 `.json` 格式储存在本地，支持多人数据隔离（特别适合健身搭子群体）。

\* \*\*🚀 冷启动防呆机制：\*\* 首次使用新动作时，自动引导录入起始重量和专属基准容量。

\## 📖 核心算法图解 (How it works)

本程序对每个动作维持一个独立的状态判定：

    状态 A (加阻力)： 达成基准容量（如 3组x8个）且有余力，下次训练直接增加目标重量。

    状态 B (加次数)： 加重后卡壳。系统锁定重量，要求下次训练总次数增加。当总次数突破阈值，退回状态 A 并加重。

    状态 C (加组数)： 次数也无法突破。系统锁定重量和次数，强行增加一组训练容量（如加至 4 组）。扛过后退回状态 A。
